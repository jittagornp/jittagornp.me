เป้าหมายในชีวิต

เป็นโปรแกรมเมอร์ ที่สร้างผลงานเจ๋งๆ ระดับโลกให้คนทั่วโลกใช้ และมีบริษัทซอฟต์แวร์เป็นของตัวเอง

ผมรู้ใจตัวเองตั้งแต่ก่อนเรียนจบ ตอนนี้อยู่ในช่วงสะสมประสบการณ์ครับ แน่นอนว่ามันทำให้ผมรักในอาชีพนี้มากๆ เพราะเรารู้ว่าตัวเองต้องการอะไรในชีวิต

แล้วใช้ tools ต่างๆ ทำอะไรบ้าง

ตอนนี้ผมทำผลงานตัวเองอยู่ครับ โดยเขียน code เองทั้งหมดเลย ตั้งแต่หน้าบ้าน (front end) ยันหลังบ้าน (back end) design database ถ่ายภาพ แต่งภาพ photoshop เองทั้งหมดครับ

หน้าบ้านผมใช้ angularjs หลังบ้านเป็น SpringMVC และ database เป็น Mysql เขียน code แบบ stateless คือไม่ใช้ session (ใช้ token แทน) เพราะต้องการการ scale-out ที่ใหญ่มากๆ ไม่ว่าจะเอาไป run บน computer กี่เครื่องๆ ก็สามารถทำงานได้สบายๆ ครับ มาดูรายละเอียด (details) แต่ละส่วนกัน

Front end (isomorphic)

angularjs

ผมเขียนใน mode html5 ครับ คือใช้ push state ของ history api มา control การเปลี่ยน page (จริงๆ ก็ใช้แค่ ui-router ซึ่งเป็น module หนึ่งของ angularjs) ทำให้เวลาเปลี่ยน page ระบบไม่เกิดการ refresh หน้าเว็บ และที่สำคัญคือรองรับ url จริง เช่น /+jittagornp ไม่เหมือนกับ #/+jittagornp ที่หลายๆ คนเขียน เวลาที่เรา refresh browser มันก็ยังอยู่หน้าเดิม ได้ content เหมือนเดิม พูดง่ายๆ ก็คือเป็นการเขียน angularjs ที่รองรับการทำ SEO (Search Engine Optimization) นั่นเอง

lesscss

ตัว css ผมใช้ less เข้ามาช่วยอำนวยสะดวกในการเขียน code เพื่อให้เราสามารถเขียน css ได้ง่ายขึ้น สะดวกขึ้น ที่สำคัญคือ reuse ใช้ของเดิมได้ (เขียนเป็น function และตัวแปรต่างๆ เพื่อเรียกใช้งานในภายหลัง)

jasmine.js

เวลาเขียน code แล้วมี test ครอบ นี่มันทำให้เรามั่นใจในการเขียน code เอามากๆ กล้าเปลี่ยน กล้าแก้ กล้ารื้อ โดยไม่กลัวอะไรเลย ต่อให้กระทบ ก็ยังรู้ว่ากระทบส่วนไหน และต้องแก้ยังไงให้ logic กลับมาถูกเหมือนเดิม ซึ่งผมก็ใช้หลัก TDD เข้ามาช่วย เวลาที่ต้องเขียน logic อะไรที่มันซับซ้อนๆ / ยาก คิดไม่ออกนี่ร่าง tdd ก่อนเลย แล้ว logic มันจะวิ่งตามทางของมันเองครับ

grunt.js
  • ไว้รวมไฟล์ (เพราะผมเขียน angularjs เป็น module ละ 1 ไฟล์)
  • ไว้ run task ต่าง ๆ ที่เราต้อง run บ่อยๆ เช่น automate test (jasmine)
  • ไว้บีบอัด/เข้ารัหัส (compress) ไฟล์ hml/css/js เป็นต้น
